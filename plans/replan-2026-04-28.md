## Strategic arc

- **v0.2** (Phases F + G + H) — closes the joint-modelling gap. This is
  what separates "Bayesian GLMM library" from "R-INLA equivalent." Without
  multi-likelihood + `Copy` + projector composition, roughly half of
  R-INLA's published case studies cannot be reproduced.
- **v0.2.1** (Phases I + J) — widens the likelihood/component zoo to cover
  the textbook cases.
- **v0.3** (Phases K + L + M) — workflow tooling, user-defined extension
  hooks, and SPDE maturity.
- **v0.4** (Phases N + O + P) — polish lap.

Estimated cadence at observed v0.1 pace: ~9–14 months to v0.2, ~18–24
months to v0.3. Acceleration depends on resourcing — see "Resourcing &
risks" below.

## Phase G — Joint-models scaffolding (8–12 weeks)

**Goal.** The R-INLA `inla.stack` + multi-likelihood + `copy` triangle.
Without this, joint longitudinal-survival, multi-response geostatistics,
and disease mapping with multiple sentinels are out of reach. Phase G is
the single highest-value missing capability and the largest architectural
change in the v0.2 horizon.

### Scope

- [ ] **Multi-likelihood support.** Promote `m.likelihood::AbstractLikelihood`
      to `m.likelihoods::Tuple{Vararg{<:AbstractLikelihood}}` keyed by
      observation index via a new `LikelihoodMap` struct. Likelihood
      derivatives in `inference/laplace.jl` already factorise over rows;
      change is in the constructor + accessors + log-density routing.
      Single-likelihood constructor remains available with a deprecation
      window of two minor releases.
- [ ] **`AbstractObservationMapping` seam.** Promote the projector field to
      its own abstract type (already noted in
      [`architecture.md`](architecture.md#the-projector-question--open-issue)
      as an ADR candidate). Concrete subtypes:
  - `IdentityMapping` — `A = I`.
  - `LinearProjector(A)` — wraps the current `m.A`. The default; v0.1
    users see no change.
  - `StackedMapping(blocks::Tuple{<:Pair{<:AbstractLikelihood,<:AbstractObservationMapping}})` —
    one mapping per likelihood block.
  - `KroneckerMapping(A_space, A_time)` — for separable space-time
    (also serves Phase M).
- [ ] **`Copy` component.**
      `Copy(target::AbstractLatentComponent; β = 1.0, hyper_β = nothing)`
      shares `target`'s latent into a new linear predictor with optional
      scaling hyperparameter. Q is the target's; the projector adds
      `β · A_target`. Hyper count = target's + (1 if `β` is estimated). No
      duplication of the latent vector — a new `LinearPredictorTerm` type
      tracks the multiplicity.
- [ ] **First joint vignette.** Reproduce R-INLA's
      [Baghfalaki et al. tutorial](https://r-inla.org/examples/Baghfalaki_et_al/Baghfalaki-et-al-example.html)
      — Gaussian longitudinal arm + Weibull/exponential survival arm +
      shared random effect via `Copy`. Survival likelihood lands in
      Phase H; vignette merges once both phases close.
- [ ] **ADR for the projector seam.** Captures the trade-off between
      explicit `AbstractObservationMapping` dispatch vs. carrying multiple
      `A`-matrices on the model. Fixes the "open issue" line in
      `architecture.md`.

### Acceptance

- R-INLA Baghfalaki et al. fit reproduces within 5 % / 10 % (means /
  hyperparameters) on a committed JLD2 oracle.
- v0.1 single-likelihood example tests still pass with no source changes
  (only deprecation warnings if the user touches the deprecated
  constructor signature).
- New ADR (next sequential number after ADR-014) accepted and merged.

### Risks

- **Scope creep.** Phase G touches the projector, the Laplace inner loop,
  the constraint stacker, the integration loop, the marginal accessors,
  and every test in `test/oracle/`. Mitigation: ship multi-likelihood
  *before* `Copy` (multi-likelihood as one PR series, `Copy` as a follow-up
  series), and accept that v0.2-alpha ships with multi-likelihood only.
- **Solo-developer review.** Phase G is the first phase where a wrong
  abstraction will compound across the rest of the v0.2 horizon. See
  "Resourcing & risks" — at least one outside reviewer for the projector-
  seam ADR is a prerequisite to merging it.

## Phase I — Multivariate IID + measurement error + replicate/group (4–6 weeks)

**Goal.** A user-visible cluster of small features that together account
for a disproportionate share of published case studies — meta-analysis,
joint models, longitudinal mixed effects, measurement-error regression.

### Scope

- [ ] `IIDND{N}` family — N-dim multivariate IID per block.
      Hyperparameters: `N` precisions + `N(N−1)/2` correlations,
      parameterised via `atanh ρ` for `N=2`, lower-triangular Cholesky for
      `N≥3`. Concrete: `IID2D`, `IID3D`, with the dispatched general form
      for higher `N`.
- [ ] **Port `BivariateIIDModel` from `IntegratedNestedLaplace.jl`** as the
      `N=2` numerical kernel — its `loggamma + atanh-ρ-Gaussian` prior is
      exactly R-INLA's `f(., model="2diid", param=c(...))` parameterisation.
      Port via the graduation pipeline (see "Numerical inheritance" below).
- [ ] `Wishart` and `InvWishart` priors for the joint precision matrix
      (drop-in for `IID2D`/`IID3D`/...).
- [ ] `PCCor1`, `PCCor0` priors (PC for AR1/AR2 correlations, Sørbye-Rue
      2017).
- [ ] `Generic2` component (covariance-parameterised; needed for
      measurement-error wrappers).
- [ ] `MEB(component, σ²_u)` — classical measurement error in covariate.
- [ ] `MEC(component, σ²_u)` — Berkson measurement error.
- [ ] `Replicate(component, n)` and `Group(component, group_id)` —
      replicated / grouped random effects sharing hyperparameters.
      Implementation is `Q = blockdiag(...)` + projector duplication; the
      load-bearing piece is making the prior be on a single shared θ
      block, not `n` copies.

### Oracle fixtures

- R-INLA `iid2d` synthetic with Wishart prior.
- R-INLA classic measurement-error regression (Carroll et al. example).
- Replicated AR1 (longitudinal panel, synthetic).

## Phase K — Posterior tooling (4–6 weeks)

**Goal.** Workflow ergonomics that the modern Bayesian community now
expects. Each item is a small PR; phase total is light by feature count
but high by visibility.

### Scope

- [ ] `posterior_sample(model, res; n_samples)` — joint draws of
      `(x, θ) ∼ π̂(· | y)` from the CCD/grid mixture. Returns
      `MCMCChains.Chains`. At each θ-node draw `θ_k` with probability
      `w_k`, then draw `x | θ_k` from the Gaussian Laplace at that node.
- [ ] `posterior_predictive(model, res, x_new)` — predictive distribution
      at new covariates / new mesh nodes; uses the projector seam from
      Phase G.
- [ ] **PSIS-LOO** (Vehtari et al.) via `PSIS.jl` — supplements `cpo`.
- [ ] `pp_check(model, res, y_obs)` with `Makie` recipes via the existing
      `LGMMakieExt`.
- [ ] `refine_hyperposterior(res; n_grid, span)` — R-INLA's `inla.hyperpar`
      equivalent. Refits the θ-posterior on a denser, skewness-corrected
      grid. Mitigates heavy-tail undersampling (the failure mode
      IntegratedNestedLaplace.jl documented for Brunei).

### Acceptance

- `posterior_sample` from Scotland BYM2 reproduces the marginal
  mean/variance summaries within MC error.
- PSIS-LOO matches R-`loo`'s elpd within 2 nats on three oracle datasets.
- `refine_hyperposterior` closes the τ posterior-mean gap on the Brunei
  fixture (see Phase L for the full-Laplace alternative).

## Phase L — User-defined components: `rgeneric` / `cgeneric` (4–6 weeks)

**Goal.** R-INLA's most-used extension hook. Whatever shape lands here
determines how easy it is for downstream users to add their own latent
models — i.e. how much of the long tail of R-INLA's component library
(`crw2`, `besag2`, `besagproper`, `clinear`, `z`, `ou`, the dozens of
user-defined components in published papers) we can avoid implementing
ourselves.

### Scope

- [ ] `UserComponent(callable; n, n_hyper)` where
      `callable(θ) → (Q::SparseMatrixCSC, log_prior::Real,
                       constraint::AbstractConstraint, log_normc::Real)`.
- [ ] Dispatch through the same `precision_matrix` / `log_hyperprior` /
      `constraints` / `log_normalizing_constant` interface — *no* changes
      to `inference/laplace.jl` if the abstract contract is right. This
      is the implicit quality gate on Phase G's abstractions.
- [ ] `cgeneric`-equivalent: same interface but with C-callable
      callbacks for performance-sensitive cases. v1: Julia-only; C
      bindings deferred to v0.4.
- [ ] **Tutorial.** Implement a non-standard latent (candidate:
      Mann–Whitney rank Markov field; or a custom matrix-exponential AR;
      or `crw2` as a user-defined component to exercise the interface).
- [ ] **Full-Laplace as a built-in `UserMarginalStrategy`.** R-INLA's
      `strategy="laplace"` is a per-`x_i` refitted Laplace; this is the
      gold-standard fix for sharply non-Gaussian latents (the Brunei
      pathology). Land as a separate `AbstractMarginalStrategy` subtype
      using the same dispatch the `UserComponent` interface validates.

### Acceptance

- Tutorial reproduces an R-INLA `rgeneric` published example numerically
  within Phase H tolerances.
- Brunei oracle (porting Phase F's tightened tolerances) closes with
  `MarginalStrategy.FullLaplace`.

## Phase M — SPDE expansion (8–12 weeks)

**Goal.** Geostatistics is one of two flagship R-INLA workflows; close
the remaining gaps versus R-INLA's `inla.spde2.matern` / `inla.spde2.pcmatern`
surface.

### Scope

- [ ] **1D SPDE** — Matérn on intervals; useful for time series with
      continuous-time gaps. Trivial reuse of the assembly code; the only
      novelty is a 1-simplex mesh constructor.
- [ ] **Non-stationary SPDE** — basis-expanded `log κ(s)`, `log τ(s)`.
      Port `NonStationarySPDEModel` from IntegratedNestedLaplace.jl (the
      numerical kernel) and redesign the prior to match R-INLA's
      `inla.spde2.matern`'s `B.kappa` / `B.tau` matrix interface.
- [ ] **Fractional-α SPDE** (α ∉ {1, 2}) via Bolin–Kirchner 2020 rational
      approximation. Marked v0.3 in
      [`packages/INLASPDE.jl/plans/plan.md`](../packages/INLASPDE.jl/plans/plan.md);
      promote to v0.3 release as part of Phase M.
- [ ] **Separable space-time.**
      `KroneckerComponent(spatial::AbstractLatentComponent, temporal::AbstractLatentComponent)`.
      Q via Kronecker, projector via `KroneckerMapping` from Phase G,
      factorisation via separable Cholesky. Composes with `LinearSolve.jl`
      and `KroneckerOperators.jl`-style operators.
- [ ] **Mesh utilities maturity.** `inla_mesh_2d` to handle non-convex
      domains end-to-end (currently flagged "fmesher-equivalent on convex
      domains" in `INLASPDE.jl/plans/plan.md`).

### Oracle fixtures

- Lindgren-Rue-Lindström 2011 §3.2 non-stationary SPDE example.
- Cameletti et al. air-pollution space-time vignette.
- 1D Matérn time series synthetic.

## Cross-cutting — Numerical inheritance from `IntegratedNestedLaplace.jl`

The companion research repo at `IntegratedNestedLaplace.jl` carries
working numerical kernels for several techniques INLA.jl will need.
Disposition is "research scratchpad → port via graduation pipeline" (see
"Resourcing" below). Concrete candidates, in port order:

1. `BivariateIIDModel` → Phase I `IID2D` numerical kernel.
2. `NonStationarySPDEModel` + the SPDE PD-failure safety net → Phase M
   non-stationary kernel.
3. **Constraint-projected simplified-Laplace mean correction**
   (`Δx ∝ ½ H⁻¹ Aᵀ (h⁽³⁾ ⊙ σ²_η)` projected onto the constraint set) —
   already partly in INLA.jl as opt-in; port the constraint-projection
   step. Phase F if cheap, Phase L otherwise.
4. **Asymmetric per-axis skewness corrections** (`stdev_corr_pos` /
   `stdev_corr_neg`) for the integration grid — drop-in for the existing
   `Grid` integration scheme. Phase K, alongside `refine_hyperposterior`.
5. **Edgeworth correction** to `log π̂(y|θ)` (4th-derivative + 3rd-derivative
   cross terms) — useful as a cheap sanity check before reaching for
   full Laplace. Phase L candidate as
   `MarginalLikelihoodCorrection.Edgeworth`.
6. **Monte-Carlo importance-sampling correction** — Phase L candidate as
   `MarginalLikelihoodCorrection.ImportanceSampled`.

**Graduation pipeline.** When a technique in `IntegratedNestedLaplace.jl`
reproduces an R-INLA fixture at tight tolerance, open a tracking issue in
`INLA.jl` titled "Port `<technique>` from INLAExperiments at commit `<sha>`",
port as the appropriate trait subtype, copy the fixture across, archive
the prototype branch.

## Cross-cutting — ADR candidates

Phases F–P generate several decisions worth recording:

- **ADR-015** — `AbstractObservationMapping` seam (Phase G prereq).
- **ADR-016** — Multi-likelihood representation: tuple of likelihoods +
  per-row `LikelihoodMap` vs. likelihood-by-block stacking (Phase G).
- **ADR-017** — `Copy` semantics: shared-latent-by-reference vs.
  shared-latent-by-projector-multiplicity (Phase G).
- **ADR-018** — Censoring representation: per-row enum vs. separate
  `CensoredLikelihood` wrapper (Phase H).
- **ADR-019** — `UserComponent` callback signature (Phase L). The
  load-bearing decision: do we require the callback to return Q + log_prior
  + constraint + log_normc as a tuple, or do we expose four separate
  methods on the user-defined struct? R-INLA's choice (the four-method
  form) trades verbosity for testability.
- **ADR-020** — Categorical likelihood parameterisation: independent-Poisson
  vs. stick-breaking (Phase J).

## Resourcing & risks

This is a 9–14-month plan to v0.2 at observed solo-developer pace.
Real options to compress:

- **Recruit reviewers, not just contributors.** The Edinburgh / KAUST /
  NTNU R-INLA group (Lindgren, Rue, Riebler) and the Julia stats
  community (Pfiffer, Fjelde, Axen, Werder, Ge) are the highest-leverage
  reviewers for the projector-seam ADR. Phase G's projector seam is the
  one place where a wrong abstraction compounds across the rest of v0.2.
- **Funding.** CZI EOSS, NumFOCUS small dev grants, NRFR infrastructure
  grant tied to NTNU. Conversion rate on (c) is highest given geography
  and existing relationships with the R-INLA group.
- **Scope narrowing.** Phase G can ship multi-likelihood without `Copy`,
  ship `Copy` without measurement-error components, ship measurement-error
  without joint longitudinal-survival. The slowest version of progress is
  still progress; the fastest version of stuck is one giant branch.

### Risks register

- **Phase G abstraction lock-in.** A wrong `AbstractObservationMapping`
  abstraction compounds. Mitigation: ADR-015 reviewed by at least one
  outside reviewer before merge.
- **Phase L extension-point breakage.** `UserComponent` is the implicit
  quality gate on every other abstraction in the package. Mitigation:
  Phase L cannot land until at least three concrete components from
  Phases F–J have been implemented as `UserComponent` instances and pass
  oracle tests. If they don't, the abstractions in F–J need more work
  before L can ship.
- **`@test_broken` regression.** A Phase F fix that makes a previously-
  broken test pass might break a previously-passing one. Mitigation: CI
  runs the full oracle suite on every PR.
- **LLM-assisted-numerics drift.** The IntegratedNestedLaplace.jl
  read-through showed where heavy LLM assistance on Laplace inner-loop
  numerics goes wrong. Use AI assistance for test scaffolding, fixture
  conversion, vignette prose, type-stability sweeps; not for the Newton
  step, the constraint stacker, or the marginal-likelihood formula.

## Working principles for this replan

Carried forward unchanged from `replan-2026-04.md`:

- One feature/phase per PR; each PR keeps the oracle suite green for
  closed phases.
- All R reference fixtures pinned to a documented INLA version; bumps
  are intentional commits with explanatory messages.

  incompatible, update the JSON / JLD2 fixture intentionally.

  release script (deferred from the 2026-04 replan; lands inside
