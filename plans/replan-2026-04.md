## Open follow-ups carried into Phase D

These are the `@test_broken` items currently tracked on `main`. None
block rc1, but each owes either a fix or a doc upgrade so the
divergence is captured outside the test file.

**Latent Gaussian Models (7 items):**

1. **Pennsylvania / Generic0 / Generic1 / Scotland-BYM2 / Gamma mlik
   offset (5 tests).** Julia's `log_marginal_likelihood` sits ~0.75
   nats per observation below R-INLA's integration estimate
   (≈ -43 nats on Scotland n=56, ≈ -50 nats on Pennsylvania n=67;
   matching shape on Generic0/1 and Gamma synthetics). Suspected:
   a missing constant in the Laplace marginal-likelihood normalizer,
   or a difference between R-INLA's "Gaussian" vs "INLA" mlik methods.
   Owner: log-marginal investigation, ~3 days; tolerances re-tighten
   to 5% rel once the offset is identified.
2. **Scotland BYM τ_b precision.** Per-CC vs global Sørbye-Rue scaling
   (Freni-Sterrantino 2018) on a 4-CC graph. Either implement per-CC
   scaling or document the convention divergence in
   `plans/defaults-parity.md`.
3. **Seasonal τ_lik.** Constraint-convention mismatch: ours applies
   `s-1` null-space-fixing constraints (Laplace contract requires
   `null(Q) = range(Cᵀ)`); R-INLA applies one sum-to-zero and lets
   the likelihood identify the residual `s-1` directions. Either
   relax the Laplace contract (allow `null(Q) ⊋ range(Cᵀ)`) or update
   the Seasonal component to expose R-INLA's constraint mode behind
   a kwarg with the conformant default.

## Risks not yet allocated

- **Selected-inversion edge cases at scale.** ADR-012 validated on
  band / random-SPD / Laplacian matrices. Real SPDE precisions on
  10⁴-vertex meshes have not been stress-tested. If
  `SelectedInversion.jl` has a memory or numerical surprise at that
  scale, Phase D slips by the effort to drop in a native Takahashi.
- **R-INLA version pinning.** `renv.lock` in `scripts/generate-fixtures/`
  pins R packages but not R itself. A user regenerating fixtures on a
  different R minor version may hit BLAS-driven numerical drift that
  fails tamper checks. Document explicitly or add a CI-only R version
  pin.
- **Matrix market for SPDE α=1.** v0.1 ships α=2 as the R-INLA default;
  α=1 is mathematically supported and tested but has no oracle fixture.
  Low priority but named so it's not forgotten.
