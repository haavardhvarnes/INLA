## Baseline

**GMRFs.jl (~M1–M6 landed)** — graph, precision, sampling, log-density,
constraints, factorization cache, nullspace, marginal variances via

oracle fixture for `qinv_rw2`. Reference-impl still present as `:dense`

**LatentGaussianModels.jl (~M1–M5 landed)** — components (Intercept,
IID, RW1, RW2, AR1, Besag, BYM2); likelihoods (Gaussian, Poisson,
Binomial); priors (PC, BYM2 φ); inference (EmpiricalBayes, Laplace with
hard constraints, INLA outer loop); integration schemes
(CCD/Grid/GaussHermite); posterior marginals + R-INLA-shaped
accessors; diagnostics (DIC/WAIC/CPO/PIT); Scotland BYM2 oracle;
simplified-Laplace skew correction (in tension with [ADR-006](decisions.md)).

**INLASPDE.jl (~M1–M6-A landed)** — FEM assembly (C, G₁, C̃, G₂),
mass lumping, precision for α∈{1,2} ([ADR-013](decisions.md)),
DelaunayTriangulation mesh generation, boundary handling, MeshProjector
(sparse barycentric), PC-Matérn prior, SPDE2 component, Meuse oracle
+ fmesher parity fixtures, GeoInterface + Makie extensions.

**INLASPDERasters.jl (landed)** — raster→mesh extraction (bilinear +
nearest), predict + quantile rasters, Makie extension.

**Scaffold-only** — `LGMFormula.jl`, `LGMTuring.jl`, `GMRFsPardiso.jl`.

## Immediate blockers (Phase B0 — before any new feature work)

Three follow-ups named in [ADR-014](decisions.md). All three land before
Phase B proper starts:

1. **[ADR-006](decisions.md) vs simplified Laplace (`85db314`).** The
   correction shipped while the ADR deferred it to v0.3. Resolve now:
   either amend ADR-006 with a superseding note recording why it came
   forward, or revert the feature behind a `strategy = :simplified`
   opt-in flag. Recommendation: amend — the correction is already
   exercised by [test_simplified_laplace.jl](packages/LatentGaussianModels.jl/test/regression/test_simplified_laplace.jl)
   and removing it loses real work.
2. **Fixture pipeline in CI.** `scripts/generate-fixtures/{gmrfs,lgm,spde}/*.R`
   exist but are not invoked by any CI workflow. The `.jld2` files in
   `test/oracle/fixtures/` are manually produced. Wire a
   `.github/workflows/regenerate-fixtures.yml` that runs on the pinned
   R-INLA version, produces the fixtures, and compares against the
   committed ones as a tamper check. No re-commit on mismatch — fail
   loud.
3. **Branch protection rules on `main`.** Named in
   [`initial-commits.md` §"Branch protection rules"](initial-commits.md)
   but not verified in effect. Required: PR-only updates, one
   approving review, `test` workflow green, linear history. Without
   this the ADR-014 anti-pattern recurs.

## Risks not yet allocated

- **Selected-inversion edge cases at scale.** ADR-012 validated on
  band / random-SPD / Laplacian matrices. Real SPDE precisions on
  10⁴-vertex meshes have not been stress-tested. If
  `SelectedInversion.jl` has a memory or numerical surprise at that
  scale, Phase D slips by the effort to drop in a native Takahashi.
- **R-INLA version pinning.** `renv.lock` in `scripts/generate-fixtures/`
  pins R packages but not R itself. A user regenerating fixtures on a
  different R minor version may hit BLAS-driven numerical drift that
  fails tamper checks. Document explicitly or add a CI-only R version
  pin.
- **Matrix market for SPDE α=1.** v0.1 ships α=2 as the R-INLA default;
  α=1 is mathematically supported and tested but has no oracle fixture.
  Low priority but named so it's not forgotten.

## What this replan does not schedule

These are deferred beyond v0.1 and tracked only to prevent scope creep:

- Space-time Kronecker components.
- Joint likelihoods / misaligned multi-response models.
- Simplified Laplace full Rue-Martino correction (beyond the skew
  term already landed).
- Fractional-α SPDE (Bolin-Kirchner rational).
- GPU factorization via CUDSS.
- `inla.rgeneric` migration guide (the architecture makes it automatic;
  a guide is still owed).
